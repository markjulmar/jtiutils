JTIUtils Class Library V3.3.0

The JulMar Utilities (JTIUtils) library is a C++ class library with a collection of useful classes and structures.  It has been collected and extended over the years to have common things that we have found useful in our product development.  This library is usable with Microsoft Visual C++ .NET (any version) but has not been tested with any other compiler.  The following is a list of the files which are present in the library, along with a short description.  Click on the filename to get a source code listing of that particular file.

 

JTIUtils.h

The primary header file for the library – this creates the linkage record and defines some base types and classes.

stlx.h

A set of useful STL functors and algorithms for use with Windows.

AdoConn.h

This implements a wrapper class around Microsoft’s ADO library for generating disconnected recordsets.

Base64.h

This implements a set of classes which will MIME-encode and decode data (base 64)

binstream.h

This implements a binary iostream class which can be used to persist data in a binary form.

CommandLineParser.h

Command line parser and tokenizer

comutls.h

COM specific classes and utility functions

DateTime.h

A generic Data/Time and Time Span class which can be used independent of MFC or ATL.

Delegates.h

Implementation type safe callbacks which can be either instance methods or static functions.  This is based on .NET  delegates.

Dyncreate.cpp

A replacement for the MFC DYNCREATE facility which uses the native RTTI support.

dyncreate.h

A replacement for the MFC DYNCREATE facility which uses the native RTTI support

EventLog.h

A set of classes which will manipulate the Windows NT/2K/XP event log, very similar to .NET.

FileEventLogger.cpp

A file-based logging class which works with the TraceLogger implementation of this library.

FileEventLogger.h

A file-based logging class which works with the TraceLogger implementation of this library

FileSystemWatcher.cpp

A class which monitors file/directory changes on NT/2K/XP.  This is based on the same class in the .NET framework.

FileSystemWatcher.h

A class which monitors file/directory changes on NT/2K/XP.  This is based on the same class in the .NET framework.

Lock.h

A set of locking primitives for managing thread locks.

Longevity.cpp

A class which enforces specific destruction ordering for global variables.

Longevity.h

A class which enforces specific destruction ordering for global variables.

ManagementObject.h

A wrapper class for the Windows Management Instrumentation interface.  This is based on the same class in the .NET framework.

MemoryMappedFile.h

A wrapper class for Memory Mapped Files.

MemPool.h

A small memory pool allocator which can be used to quickly allocate small objects.

memstream.h

An implementation of the binstream class which uses a memory store as it’s backing.

MsxmlHelper.h

A wrapper around the MSXML parser.

Observer.h

A simple observer list which allows notifications to occur to any object contained in the list.

PsList.h

A set of classes which allow interaction with the Process list under NT/2K/XP based on the .NET framework implementation.

RefCount.h

Support for reference counting and the pimpl or handle-envelope pattern.

Registry.h

A set of registry access classes which are very similar to the .NET framework implementation.

RWLock.h

A multi-reader/single-writer locking structure.

SEHException.h

A simple class which allows SEH exceptions to be caught as standard C++ exceptions.

ServiceSupport.cpp

A set of base classes which implement NT service control logic for creating NT services.

ServiceSupport.h

A set of base classes which implement NT service control logic for creating NT services.

SingletonRegistry.h

A class for managing singletons.

sqlstream.cpp

A set of SQL creation classes which allow the building of stored procedure invocations to be done with iostream operators.  This simply generates the SQL command text, it does not actually invoke the SQL commands directly.  These classes were based on the patterns described by Microsoft for n-tier development when using COM.

sqlstream.h

A set of SQL creation classes which allow the building of stored procedure invocations to be done with iostream operators.  This simply generates the SQL command text, it does not actually invoke the SQL commands directly.  These classes were based on the patterns described by Microsoft for n-tier development when using COM

StatTimer.h

A statistics timer class which makes timing functional processes easier.

Synchronization.h

Thread synchronization primitives.

ThreadPool.cpp

An implementation of a thread manager which uses I/O completion ports to control a set of threads.  This is wrapped further by the WorkerThreadPool class which implements a full worker thread pool implementation on top of this support.

ThreadPool.h

An implementation of a thread manager which uses I/O completion ports to control a set of threads.  This is wrapped further by the WorkerThreadPool class which implements a full worker thread pool implementation on top of this support.

ThunkWriter.h

This class writes out x86 thunks which can be used to generate dynamic callbacks.

Timers.cpp

This implements a timer class which uses a dedicated thread for calling back to a set of clients on a programmable basis.

Timers.h

This implements a timer class which uses a dedicated thread for calling back to a set of clients on a programmable basis.

TraceLogger.cpp

This implements a type-safe trace logging facility which allows for observers to monitor the trace output.

TraceLogger.h

This implements a type-safe trace logging facility which allows for observers to monitor the trace output.

tscontainer.h

This implements a type-safe container which locks/unlocks on access.

WorkerThreadPool.h

This implements a worker thread pool implementation that allows work to be scheduled into the thread pool to occur on a background thread.  The startup/shutdown of the worker threads can be altered via trait classes.

XmlConfig.h

This implements a XML version of .INI file profiles through a set of get_xxx and set_xxx methods that extract specific types from the given XML file.

XmlParser.cpp

This implements a poor-mans XML parser which can be used when MSXML is not available.

XmlParser.h

This implements a poor-mans XML parser which can be used when MSXML is not available.
